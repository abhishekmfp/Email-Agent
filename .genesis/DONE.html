1 · The Locked Spec (what we are building)

Cognitive job: The AI understands human intent and transforms an imprecise natural-language request into a complete, structured, high-quality email draft (recipients, subject, body, tone, purpose) — asking concise clarification when factual info is missing. The LLM performs cognition only; it never decides to send, triggers Gmail, authenticates, enforces rules, or owns workflow state. Those are deterministic application logic.

Inputs: a single free-text user request per interaction (untrusted, validated & sanitized before the LLM sees it) + optional one-time user profile; startup-time config/credentials (LLM API key, model, Gmail OAuth tokens, app config) from env/secure config — never per-request.  →  Outputs: a strongly-typed structured draft object (the AI↔app contract) → rendered preview for human approval → after explicit approval, a real Gmail send returning message/thread ID + success response, or a structured error confirming nothing was sent.

Autonomy level: Level 1 (Assistive AI) — cognitive tasks only, zero operational authority.  |  Human-in-the-loop points: explicit approval required before EVERY Gmail send (including edited/regenerated/retried drafts); the delivered email must be byte-identical to the approved version; reject/edit supported; LLM can never bypass the approval gate.

Failure tolerance: Priority Safety > Correctness > Transparency > Reliability > Automation. Wrong recipient / PII / false commitment = catastrophic, prevented by HITL. Transient infra errors: conservative exponential-backoff retries then surface. Uncertain delivery state: report & never auto-resend. Every LLM response schema-validated before the approval workflow; malformed output never presented as approvable.

Trust boundary: Four zones — External(untrusted) / Application(trusted, owns all side effects) / LLM(restricted-trust, credential-blind) / Gmail(external, reached only post-approval). Prompt-injection→exfiltration is a modeled threat, defended by data minimization (secrets withheld from the LLM) + structural message-role separation + output validation. Recipients are never inferred; the approved recipient list is part of the immutable final draft.
